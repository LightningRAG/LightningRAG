### 贡献指南

**English:** [CONTRIBUTING.md](./CONTRIBUTING.md)

#### 1 Issue 规范

- Issue 仅用于缺陷报告、功能请求与设计相关讨论；其他类型问题可能被直接关闭。使用本项目时的一般疑问，请优先通过项目 Discussions、Issue 模板或与维护者约定的渠道求助。

- 提交 Issue 前，请先搜索是否已有相同或类似问题。

#### 2 Pull Request 规范

- 将本仓库 Fork 到你自己的账号下，**不要**在本仓库直接新建长期分支。

- 提交信息建议格式：`[文件名]: 变更说明。`（例如 `README.md: 修复某问题`）

- **请向 `develop` 分支提交 PR，不要指向 `master`。**

- 若 PR 修复缺陷，请说明关联问题或复现方式。

- 合并 PR 通常需要维护者侧评审流程：一人审核通过后，再由另一人复核并合并（以项目实际流程为准）。
